# 我的知识库 — PARA 系统

这是一个基于 **PARA 方法**（by Tiago Forte）组织的 Obsidian 知识库。PARA 将所有数字信息归入四个顶级分类，让你在几秒钟内找到任何内容。

---

## 文件夹结构

```
obsidian_vault2/
├── Projects/        # 正在进行的项目（有明确完成条件）
├── Areas/           # 长期责任领域（持续维护，无截止日期）
├── Resources/       # 感兴趣的主题与参考资料
├── Archives/        # 不再活跃的内容归档
│   ├── Projects/
│   ├── Areas/
│   └── Resources/
├── Daily/           # 每日笔记（YYYY-MM-DD.md）
├── Clippings/       # 网页剪藏
└── Templates/       # 笔记模板
```

---

## PARA 四大分类

### Projects — 项目
**定义**：有明确目标、有预期完成时间的工作单元。

- 每个项目是一件可以被"完成并划掉"的事
- 示例：完成网站改版、写完季度报告、规划家庭旅行

### Areas — 领域
**定义**：需要长期持续关注和维护的责任范围，没有终点。

- 领域定义的是一个持续的标准，而非一个具体目标
- 示例：健康管理、财务规划、职业发展、写作

### Resources — 资源
**定义**：感兴趣的主题或将来可能有用的参考材料。

- 按主题组织，不需要与当前工作直接相关
- 示例：AI 研究、设计参考、读书笔记、工具收藏

### Archives — 归档
**定义**：前三类中不再活跃的所有内容。

- 不删除，只归档——保留历史，不占用注意力
- 项目完成 → 移入 Archives/Projects/
- 领域停止 → 移入 Archives/Areas/

---

## 日常使用流程

1. **捕捉**：新想法、链接、信息 → 放入今日 `Daily/YYYY-MM-DD.md` 的 Inbox 区块
2. **整理**：每周将 Inbox 内容分发到 Projects / Areas / Resources
3. **查找**：按 PARA 分类直接导航，或使用 Obsidian 全文搜索
4. **归档**：项目完成或领域停止时，整体移入 Archives

---

## 核心原则

> **按行动性组织（Organize by Actionability）**
>
> 不按学科主题分类，而按"这对我当前的目标有多直接的帮助"来决定信息放在哪里。
> 越接近当前目标的信息，越靠近文件夹层级的顶端。

## 模板

`Templates/` 文件夹提供以下模板：

| 模板 | 用途 |
|---|---|
| `daily-note.md` | 每日笔记 |
| `project-readme.md` | 项目 README |
| `area-overview.md` | 领域概览 |
| `resource-index.md` | 资源主题索引 |
| `meeting-note.md` | 会议记录 |
| `book-note.md` | 读书笔记 |

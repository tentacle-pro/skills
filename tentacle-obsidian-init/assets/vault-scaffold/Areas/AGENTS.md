# Areas 文件夹 — 智能体指南

## 用途

存放**责任领域**相关内容。领域是你需要长期持续关注和维护的生活/工作方面，没有明确终点。

## 核心规则

- 每个领域独立存放于一个子文件夹
- 领域内容是**持续更新**的，不会被"完成"和划掉
- 领域可以孵化出项目：当领域内出现具体目标时，在 Projects 中新建项目
- 领域不再活跃时，整体移入 `Archives/Areas/`

## 典型领域示例

**工作类**
- 职业发展
- 团队管理
- 产品/项目管理

**个人类**
- 健康与运动
- 财务
- 家庭关系
- 个人成长

**创作类**
- 写作
- 摄影
- 内容创作

## 子文件夹建议结构

```
Areas/
  领域名称/
    overview.md     ← 领域定义、当前状态、相关项目列表
    log.md          ← 持续记录、反思
    assets/         ← 相关素材和参考
```

## 智能体操作指引

- 当信息没有明确截止点但需要长期维护时，归入此文件夹
- 定期检查各领域是否有可以具象化为项目的目标
- 问"这是否是我需要长期维护的标准或责任？" → 是则归入 Areas
- 领域停止关注后，移至 `../Archives/Areas/`

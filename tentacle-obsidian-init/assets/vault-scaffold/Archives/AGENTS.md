# Archives 文件夹 — 智能体指南

## 用途

存放**所有不再活跃的内容**。Archives 是 PARA 系统的"冷存储"，保留历史记录，但不占用日常工作的注意力。

## 核心规则

- **只进不出**原则：归档的内容通常不再移回活跃文件夹（特殊情况除外）
- 保持与来源文件夹相同的子结构，便于追溯
- 归档时在文件夹名称前加日期前缀，如 `2025-03 项目名称`
- 不要删除内容——归档而非删除

## 子文件夹结构

```
Archives/
  Projects/         ← 已完成或暂停的项目
  Areas/            ← 不再活跃的责任领域
  Resources/        ← 不再感兴趣的主题资源
```

## 何时归档

| 来源 | 归档条件 |
|---|---|
| Projects | 项目已完成、已取消或长期搁置 |
| Areas | 该领域不再是你生活/工作的一部分 |
| Resources | 对该主题已不再感兴趣或不再相关 |

## 何时"解档"

虽然罕见，但可以将内容移回活跃文件夹：
- 搁置的项目重新启动 → 移回 Projects
- 重新关注的领域 → 移回 Areas
- 重新感兴趣的主题 → 移回 Resources

## 智能体操作指引

- 不要在日常工作中主动检索 Archives，除非用户明确要求查找历史内容
- 归档操作：将源文件夹整体移入对应子目录，并重命名加日期前缀
- 归档时可在文件夹根目录添加简短 `archive-note.md` 说明归档原因和时间

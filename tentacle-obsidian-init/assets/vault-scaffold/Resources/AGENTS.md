# Resources 文件夹 — 智能体指南

## 用途

存放**感兴趣的主题、参考资料和知识素材**。Resources 是按主题组织的知识库，供将来参考和学习使用。

## 核心规则

- 按**主题/兴趣领域**组织子文件夹，而非按项目或责任
- 内容不需要与当前活跃工作直接相关——感兴趣就可以收集
- Clippings（网页剪藏）、读书笔记、学习材料等归入此处
- 不再感兴趣的主题，移入 `Archives/Resources/`

## 典型资源主题示例

- AI 与机器学习
- 生产力方法论
- 写作技巧
- 设计参考
- 编程语言/框架
- 读书笔记
- 工具与软件
- 健康与营养

## 子文件夹建议结构

```
Resources/
  主题名称/
    index.md        ← 主题概述、关键概念、推荐资源
    notes/          ← 具体笔记和摘录
    references/     ← 外部链接、书单等
```

## 与其他文件夹的区别

| 文件夹 | 问题 |
|---|---|
| Projects | 这是否服务于当前有截止日期的目标？ |
| Areas | 这是否是我需要长期维护的责任？ |
| **Resources** | **这是否是我感兴趣的主题或参考材料？** |
| Archives | 这是否已不再活跃？ |

## 智能体操作指引

- 网页剪藏、文章摘录默认归入对应主题子文件夹
- 搜索参考资料时，优先在此文件夹检索
- 新增主题时创建子文件夹，并添加简短 index.md 说明主题
- 定期回顾，将已无兴趣的主题移至 `../Archives/Resources/`

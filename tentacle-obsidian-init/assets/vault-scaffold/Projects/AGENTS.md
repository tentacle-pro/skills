# Projects 文件夹 — 智能体指南

## 用途

存放**所有正在进行的项目**。项目的定义是：有明确目标、有预期完成时间的短期工作单元。

## 核心规则

- 每个项目独立存放于一个子文件夹，命名格式：`项目名称` 或 `YYYY-MM 项目名称`
- 项目必须有**明确的完成条件**（可以被划掉的事项）
- 项目完成或暂停后，整体移入 `Archives/Projects/`
- 不要在此存放永久持续的责任领域（那属于 Areas）

## 区分 Project vs Area

| 项目 (Project) | 领域 (Area) |
|---|---|
| 完成网站改版 | 网站维护 |
| 写完季度报告 | 写作 |
| 学完某门课程 | 个人成长 |
| 规划并完成家庭旅行 | 家庭 |

## 子文件夹建议结构

```
Projects/
  项目名称/
    README.md       ← 项目目标、背景、完成标准
    notes/          ← 过程笔记
    assets/         ← 相关素材
```

## 智能体操作指引

- 新建项目时，在此文件夹下创建子文件夹并添加 README.md
- 查找当前活跃工作时，优先在此文件夹中检索
- 判断某条信息是否属于项目：问"这是否服务于一个有截止条件的目标？"
- 归档项目时，将整个子文件夹移至 `../Archives/Projects/`

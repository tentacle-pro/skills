# Clippings 文件夹 — 智能体指南

## 用途

这是 **Obsidian Web Clipper 的自动写入目录**，也是所有网页剪藏的临时落点。

**Clippings 是 Inbox，不是归宿。** 内容不应长期堆积在此。

## 核心规则

- 不要主动将新笔记创建在此文件夹
- 此文件夹的内容需要定期**分发**到 PARA 对应位置后清空
- 分发原则见下方决策树

## 剪藏内容的分发规则

```
Clippings/ 中的一篇剪藏
      ↓
  它服务于某个进行中的项目？
  ├── 是 → 移至 Projects/<项目名>/
  └── 否
        ↓
    它是某个主题的长期参考资料？
    ├── 是 → 移至 Resources/<对应主题>/
    └── 否（随手看看，无长期价值）→ 直接删除
```

## 智能体操作指引

- 用户要求保存网页内容时，**不要**默认写入此文件夹；应按上方决策树直接写入目标位置
- 如果无法判断归属，写入 `Daily/<今日日期>.md` 的 Inbox 区块，由用户决定
- 定期提示用户清理此文件夹（当文件数量积累较多时）
